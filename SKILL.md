# Lore Forge

## Overview

Lore Forge 读取用户的简历文件，自动识别其中涉及的所有技术知识点，生成一个完整的 GitBook 风格自学文档网站（单个 HTML 文件）。网站包含章节导航、知识点教学、前置知识讲解和面试题模拟。

## Workflow

### Step 1: 解析简历文件

根据文件格式选择对应的解析方式：

- **PDF**: 执行 `npx tsx scripts/parse_resume.ts <file_path>` 提取文本
- **DOCX**: 执行 `npx tsx scripts/parse_resume.ts <file_path>` 提取文本
- **TXT / MD**: 直接使用 Read 工具读取文件内容

执行脚本前，先在 skill 目录下安装依赖：
```bash
cd <skill_dir>/scripts && npm install 2>/dev/null
```

### Step 2: 分析知识点

从简历文本中提取所有技术知识点，按以下维度分类：

1. **编程语言与框架** — 如 Java、Python、React、Spring Boot 等
2. **系统与架构** — 如微服务、分布式系统、消息队列等
3. **数据库与存储** — 如 MySQL、Redis、MongoDB 等
4. **DevOps 与工具** — 如 Docker、K8s、CI/CD、Git 等
5. **计算机基础** — 如操作系统、网络、数据结构与算法等
6. **业务与领域知识** — 如支付系统、推荐系统等
7. **其他技能** — 如项目管理、软技能等

对每个知识点，还需识别其前置知识（prerequisite），形成知识依赖图。

### Step 3: 生成章节结构

按照知识分类组织章节，每个章节包含：

- 章节概述
- 各知识点的教学内容（含前置知识说明）
- 面试题模拟（每个知识点 2-5 道常见面试题）

### Step 4: 生成 HTML 文件

使用 `references/html_guide.md` 中的视觉规范和 `assets/template.html` 中的模板结构，生成最终的单文件 HTML 网站。

**关键要求：**
- 读取 `assets/template.html` 作为 HTML 骨架
- 将生成的章节内容填充到模板中
- 输出为单个 HTML 文件，保存到用户指定位置或简历同目录下

## 内容编写规范

### 知识点教学格式

每个知识点应包含：

1. **是什么** — 一句话定义
2. **为什么重要** — 在实际开发中的作用
3. **核心原理** — 用简洁语言解释工作机制
4. **前置知识** — 理解该知识点需要先掌握什么（如有）
5. **实际应用** — 简历中对应的使用场景

### 面试题格式

每道面试题包含：
- 题目（模拟真实面试官的问法）
- 参考答案（简明扼要，突出关键点）
- 追问方向（面试官可能的追问）

## 视觉风格规范

### 禁止事项
- 禁止蓝紫渐变、玻璃拟态、高透明模糊、光泽效果
- 禁止使用 emoji 图标
- 禁止卡片堆叠式统计布局
- 禁止装饰性图表

### 允许使用
- 纯色或低饱和背景色
- 统一风格的 SVG 图标
- 适度阴影建立层级关系
- 清晰边框作为辅助
- 少量强调色突出关键操作
- 充分利用横向空间，保持偏高信息密度

### 设计目标
工具型 UI，层级堆叠风格。界面应接近一个会被长期使用的工具，而非展示型页面。
